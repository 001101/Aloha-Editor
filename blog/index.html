We've been busy...

Beautiful design, boilerplate, & breaking changes

18 January, 2014 by Petro Salema

Here come our first breaking changes of 2015—yes, we've been busy!

We're releasing improvements to the way UI wiring can be done with Aloha Editor. The goal of these efforts was to make the design more comprehensible and comprehensive for API users.

One of the design principles behind Aloha Editor is the of minimal shared state.

These changes include:

  • Simplified wiring and reduced boilerplate
  • Allowing per-editable configuration

If you followed those two links above, you would have noticed that we've introduced a new API module aloha.ui, and an updated guide for how to get started that has more comprehensive content and is more beautifully designed.

These are breaking changes. View the source of our UI demo to see how the UI wiring there was able to be far less code and much simplified. Use that as a guide to update your own implementation.

We hope you will find these changes helpful.
